# w音乐首版发布合规清单

本文件用于准备 HarmonyOS 应用市场材料，不替代平台审核规则或专业法律意见。

## 已落实到项目

- 中文名称：w音乐；英文名称：wPlayer。
- 发布主体：个人开发者韩旭龙。
- 联系邮箱：wbbb0@outlook.com。
- 当前功能范围：用户主动选择本地音乐文件的纯离线播放器。
- 无账号、广告、应用内购买、统计、崩溃上传、在线更新检查或在线内容。
- 未申请 `ohos.permission.INTERNET`。
- 仅声明振动、文件持续访问和后台运行权限，并在应用内说明用途。
- 已移除备份扩展和备份配置，首版不参与应用数据系统备份。
- 已提供应用内“关于”“隐私政策”“数据与权限”“开源软件许可”页面。
- 已提供仓库级 `LICENSE`、`PRIVACY.md` 和 `THIRD_PARTY_NOTICES.md`。
- 开源许可证：GPL-3.0-only。

## 上架前仍需由发布者完成

- 申请并完成软件著作权登记；它通常是中国大陆分发的重要权属材料，但具体是否为当前渠道的强制前置条件，应以提交时的应用市场要求为准。
- 完成应用备案、实名/开发者认证及其他中国大陆分发所需手续，并以提交时平台页面为准。
- 确认“w音乐”名称、图标、截图、介绍、关键词不侵犯他人商标、著作权或其他权利。
- 准备应用图标、手机/平板/2in1 截图、短介绍、完整介绍、版本更新说明、分类和内容分级问卷。
- 在应用市场后台填写与 `PRIVACY.md` 一致的隐私政策；如平台要求公开 URL，应将该文件发布到稳定的 HTTPS 页面。
- 对发布构建做真机验收：安装/升级、导入、后台播放、系统媒体控制、清空音乐库、卸载重装，以及各支持音频格式。
- 当前 `https://github.com/wbbb0/wPlayer` 为私有仓库。正式发布前需将对应版本源码公开，建立不可变版本标签，并确保该标签的源码和构建脚本可公开取得，以履行 GPL 源代码提供义务。
- 复核最终 HAP 的权限、组件、SDK/证书、签名和第三方软件清单，且不得包含本地签名材料。

## 建议的应用市场文案

短介绍：

> 一款干净、流畅、专注本地音乐体验的 HarmonyOS 播放器。

完整介绍：

> w音乐是一款为希望获得干净、流畅且专注的本地音乐体验的用户而开发的播放器，以简洁设计、快速响应和高性能播放为目标。
>
> 用户可以通过系统文件选择器主动添加设备中的音乐，在应用内浏览歌曲与专辑、管理播放列表，并使用后台播放和系统媒体控制。应用不会提供、下载或推荐任何音乐内容。
>
> 首个公开版本完全离线，不包含账号、广告、应用内购买、使用统计或崩溃上传。音乐库、播放列表和应用设置保存在用户设备中，不会发送给开发者或第三方服务器。

建议关键词：本地音乐、音乐播放器、离线播放、播放列表、HarmonyOS。

## 未来加入 WebDAV / SMB 前

联网功能不能直接沿用本首版隐私材料。实现和发布前至少应完成：

- 增加网络权限，并更新应用市场“是否联网”、隐私标签、权限说明、应用介绍和测试账号/测试说明。
- 更新应用内及公开隐私政策，明确服务器地址、用户名、Token/密码、域、根目录、同步状态和日志的本地处理方式。
- 明确凭据所用的具体安全存储能力；“Asset Store”不是可直接写进发布材料的最终技术名称，需以实际采用并验证的 HarmonyOS 安全存储 API 为准。
- 提供“不保存密码”、删除服务器配置、清除凭据/同步缓存、移动网络和自动同步开关。
- WebDAV 明文 HTTP 与忽略证书校验必须逐服务器显式启用并醒目警告；自签名证书不等同于“连接未加密”，应准确提示为“无法由系统信任链验证身份”等风险。
- SMB 仅支持 SMB 2/3，拒绝 SMB1；不做局域网扫描；尽可能协商签名/加密并提示降级风险。
- 对上传、覆盖和删除播放列表进行确认、冲突策略和失败恢复；不得删除服务器上的音乐文件。
- 日志不得记录密码、Token、Authorization Header；用户名、服务器地址和文件路径默认脱敏。
- 重新生成第三方软件清单，特别核对 WebDAV/SMB 客户端库的许可证、版权声明、原生库和加密组件。
- 增加传输中断、证书错误、弱网络、大文件、路径编码、冲突、重复同步和凭据删除的测试。
